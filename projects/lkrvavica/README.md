## Project Overview

This project applies **graph-based methods** to the [Sakila](https://dev.mysql.com/doc/sakila/en/) open-source DVD rental database to build a **multiplex film similarity network** ‚Äî without relying on any explicit user ratings or reviews.

Two complementary similarity signals are modelled:

| Layer | Signal | Edge weight |
|---|---|---|
| **Actor layer** | Two films share cast members | IFA-weighted Jaccard similarity |
| **User layer** | Two films were rented by the same customers | Cosine co-rental similarity (threshold ‚â• 0.10) |

These two layers are combined into a **Borda-rank projected multiplex network** and analysed across eight research questions covering centrality, community structure, genre clustering, hub films, and revenue prediction.

## Dataset

**Sakila** is an open-source relational database published by MySQL as a sample schema. It models a fictional DVD rental business and contains 16 relational tables.

### Tables used in this project

| Table | Rows | Role |
|---|---|---|
| `film` | 1,000 | Film metadata ‚Äî title, genre, rating, length, rental rate |
| `film_actor` | 5,462 | Many-to-many mapping between films and actors |
| `film_category` | 1,000 | Genre assignment for each film |
| `actor` | 200 | Actor identities |
| `category` | 16 | Genre names |
| `rental` | 16,044 | Every individual rental transaction |
| `inventory` | ~4,580 | Physical film copies linking rentals to films and stores |
| `customer` | 599 | Customer identities |
| `payment` | ~16,000 | Payment records linked to rentals |
| `store` | 2 | Two store locations |

### Key data characteristics

- **No explicit ratings** ‚Äî behaviour must be inferred entirely from rental transactions
- **0% multi-genre films** ‚Äî each film belongs to exactly one genre category
- **42 films never rented** ‚Äî these appear as isolated nodes in the user network
- **Rental popularity is heavily skewed** ‚Äî a few films account for a disproportionate share of rentals
- **Static snapshot** ‚Äî data covers 2005‚Äì2006 only; no temporal dynamics

## Methodology

### 1. Network construction

#### Actor Co-occurrence Network
- **Edge condition:** two films share an edge if they share at least one actor
- **Edge weight:** IFA-weighted Jaccard similarity
  - Jaccard = |shared actors| / |union of actors|
  - IFA (Inverse Frequency Actor) down-weights prolific actors ‚Äî analogous to IDF in text retrieval
- Result: **68,919 edges**, density **0.138**

#### User Co-rental Network
- **Raw version:** edge if ‚â• 2 customers rented both films; weight = raw shared-customer count (used as baseline only ‚Äî strongly confounded with popularity, œÅ ‚âà 0.99)
- **Cosine-normalised version (used in multiplex):** weight = cosine similarity of rental vectors; threshold ‚â• 0.10
  - Cosine = shared customers / ‚àö(|renters of A| √ó |renters of B|)
  - Normalisation removes the popularity confound (œÅ drops from 0.99 to 0.37)
- Result: **29,589 edges**, density **0.059**

#### Statistical validation
Both layers validated against degree-preserving bipartite null models (20 randomised rewirings):

| Layer | Observed density | Null mean ¬± std | Z-score |
|---|---|---|---|
| Actor | 0.138 | 0.087 ¬± 0.001 | **96** |
| User (raw) | 0.324 | 0.179 ¬± 0.000 | **538** |

Both layers carry signal far beyond random expectation.

#### Multiplex combination
Two aggregation strategies are compared:

| Strategy | Method | Used for |
|---|---|---|
| **Borda rank projection** | Rank edges within each layer, sum normalised ranks | Primary combined graph |
| Linear projection | Sum raw normalised weights directly | Baseline comparison |

Borda is preferred as it avoids scale incompatibility between layers.

- Combined graph: **94,378 edges**, density **0.189**
- **95.6% of edges are unique to one layer** ‚Äî the layers are genuinely complementary
- Only **4,130 edges (~4.4%)** confirmed by both layers ‚Äî these are the strongest similarity signals
- Graph is **fully connected** (1 component across all 1,000 films)

### 2. Research questions

Eight research questions are investigated using centrality analysis, community detection, genre evaluation, and revenue modelling.

#### Centrality metrics
All metrics computed in both layers separately:

| Metric | Description |
|---|---|
| Weighted degree strength | Sum of edge weights for each node |
| Betweenness centrality | Frequency of appearing on shortest paths (bridge role) |
| PageRank | Recursive importance via neighbour importance |
| Eigenvector centrality | Connection to well-connected nodes |
| Closeness centrality | Average distance to all other nodes |

#### Community detection methods compared

| Method | Description |
|---|---|
| Louvain (each layer) | Greedy modularity optimisation on single-layer graphs |
| Louvain (combined Borda) | On the multiplex projection |
| Supra-adjacency Louvain | Coupled two-layer graph with inter-layer weight œâ ‚àà {0.1, 0.5, 1.0} |
| Cross-layer consensus | Co-assignment frequency matrix across multiple runs and both layers |

#### Evaluation metrics

| Metric | What it measures |
|---|---|
| Spearman œÅ | Rank correlation ‚Äî robust to skewed distributions |
| NMI (multi-label) | Agreement between community partition and genre labels |
| Genre purity | Fraction of a community sharing its dominant genre |
| Stability | Average NMI across 20 independent Louvain runs |
| Null-model Z-score | Standard deviations above degree-preserving random graphs |
| Partial correlation | Correlation after controlling for rental popularity |

## Key Results

### Network construction

- Both layers carry genuine structural signal (null-model Z-scores of 96 and 538)
- Raw co-rental counts are almost entirely confounded with popularity (œÅ = 0.99); cosine normalisation reduces this to œÅ = 0.37
- 95.6% of multiplex edges are unique to one layer ‚Äî the combination is well-justified

### Research question findings

| RQ | Short answer | Key evidence |
|---|---|---|
| RQ1 | Popular ‚âÝ structurally central | All centrality-popularity œÅ < 0.09; betweenness is *negatively* correlated (œÅ = ‚àí0.21) |
| RQ2 | Layers are independent | Edge Jaccard = 0.044; cross-layer rank œÅ ‚âà 0.00 |
| RQ3 | User + Louvain or consensus best | NMI: consensus 0.187 > user 0.080 > actor 0.043 |
| RQ4 | User layer far better for genre clustering | Genre purity: user 0.77 vs. actor 0.27 |
| RQ5 | No dual-layer hub films | Hub rank œÅ = ‚àí0.004; 4 quadrants each ~25% |
| RQ6 | Film attributes don't predict network position | Max œÅ = 0.065; rating Kruskal-Wallis p = 0.79 |
| RQ7 | Stores differ locally, converge globally | Store edge Jaccard = 0.019; identical community NMI (0.233‚Äì0.236) |
| RQ8 | Network position doesn't independently predict revenue | Partial correlation = 0.026 (p = 0.40) after controlling for popularity |

### Notable findings

- **Top-10 by popularity and top-10 by betweenness have zero overlap** ‚Äî bridge films are low-popularity niche titles, not blockbusters
- **Consensus community modularity (0.80) is mechanically inflated** ‚Äî confirmed by permutation test (Z = ‚àí1.6 √ó 10¬π¬≤); NMI (0.187) is the reliable metric
- **Supra-adjacency Louvain at œâ = 0.5 collapses to only 2 communities** ‚Äî inter-layer coupling is too aggressive at this value
- **Games, Animation, Music, and Children genres** are the most co-rented genre clusters in the user network

## Limitations

| Limitation | Impact |
|---|---|
| Small customer base (599 renters) | Sparse co-rental signals for niche films; 42 films have no rentals at all |
| No explicit ratings | Rental ‚âÝ satisfaction ‚Äî negative experiences are indistinguishable from positive ones |
| Static historical snapshot (2005‚Äì2006) | No temporal dynamics, concept drift, or seasonal effects |
| Binary rental events | Repeat rentals by the same customer are not weighted |
| Single-label genre evaluation | Sakila assigns one genre per film ‚Äî nuanced cross-genre similarity cannot be evaluated |
| O(n¬≤) edge construction | Pairwise comparison across 1,000 films is feasible; does not scale directly to large catalogs |
| Louvain non-determinism | Stability scores of 0.22‚Äì0.36 indicate moderate sensitivity to random seed |
| Consensus modularity inflation | The co-clustering frequency construction mechanically inflates modularity ‚Äî use NMI as the reliable metric |

## Future Work

- **Additional network layers** ‚Äî director co-occurrence, language, release year proximity, MPAA rating similarity
- **Graph embeddings** ‚Äî apply Node2Vec or GraphSAGE to learn low-dimensional film representations from the multiplex structure
- **Temporal analysis** ‚Äî slice rentals by month to track how co-rental communities evolve over the rental season
- **Scalability** ‚Äî replace O(n¬≤) pairwise construction with locality-sensitive hashing (LSH) or approximate nearest-neighbour indexing
- **Downstream evaluation** ‚Äî test whether multiplex similarity predicts held-out rentals better than single-layer baselines or popularity
- **Larger datasets** ‚Äî apply the pipeline to MovieLens or a streaming service dataset to test generalisability
- **Dynamic community detection** ‚Äî use temporal graph algorithms (TILES, evolutionary Louvain) to track community membership shifts
